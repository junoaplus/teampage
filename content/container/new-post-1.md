# CONTAINER 개요
---
## 컨테이너란?
```
호스트 OS 위에 여러 개의 격리된 환경을 만들어서 각각의 환경에서 독립적으로 프로세스와 파일 시스템 등을 실행하는 기술이다. 
```

---
## 컨테이너 이미지란?
```
컨테이너 이미지는 도커와 같은 컨테이너 가상화 기술에서 사용되는 실행 가능한 소프트웨어 패키지이다. 이 패키지는 애플리케이션을 실행하는 데 필요한 모든 소프트웨어를 포함하며, 애플리케이션의 코드, 라이브러리, 실행 환경, 시스템 도구 등이 포함될 수 있다. 컨테이너 이미지는 일반적으로 컨테이너 레지스트리에서 관리되며, 필요에 따라 다운로드하거나 업로드하여 사용할 수 있다. 컨테이너 이미지를 기반으로 컨테이너를 생성하고 실행하면, 해당 컨테이너는 이미지에 포함된 애플리케이션을 실행하게 됨.
```

---
## 컨테이너 사용 이유
```
1. 한 대의 컴퓨터에 많은 소프트웨어를 설치해서 사용할 경우 서로 영향을 미칠 수 있으므로 독립적인 컴퓨터가 필요함.
2. 해당 소프트웨어의 컨테이너 이미지를 받아서 실행하면 소프트웨어 설치를 쉽게 할 수 있음.
3. 컴퓨터 환경을 이전할 때 많은 소프트웨어를 다른 컴퓨터에 동일한 환경으로 다시 설치해서 사용해야 할 때 유용함.
4. 개발 시스템에서 서비스 시스템으로 이전(배포) • 성능이 좋은 새로운 서버로 이전(마이그레이션)이 용이함.
```


---
## 도커 컨테이너의 영구 스토리지 

1. 필요성
```
도커 컨테이너의 저장소는외부와 단절되어 있으며 컨테이너가 삭제될 때 같이 삭제됨. 따라서 컨테이너가 삭제되어도 데이터를 보관할 수 있는 영구 저장소가 필요함.
``` 
2. 볼륨 (Volume)
```
 컨테이너 자체에 볼륨을 만들어서 데이터를 저장하는 것. 볼륨은 컨테이너에서 데이터를 읽고 쓰기 위한 디렉토리이다. 이 방법은 호스트 머신의 파일 시스템과는 달리 컨테이너와 볼륨 사이의 의존성을 제거하여 이식성을 높이는 장점이 있다. 또한, 컨테이너가 다시 생성되어도 데이터가 유지되므로 데이터 손실의 위험을 줄일 수 있다.
```
3. 바인드 마운트 (Bind mount)
```
 바인드 마운트를 사용하면 호스트 파일 시스템의 특정 경로를 컨테이너로 바로 마운트할 수 있다.
```

4. 볼륨 vs 바인드 마운트
```
볼륨(volume)과 바인드 마운트(bind mount)의 가장 큰 차이점은 Docker가 해당 마운트 포인트를 관리해주느냐 안 해주느냐이다. 볼륨을 사용할 때는 우리가 스스로 볼륨을 생성하거나 삭제해야하는 불편함이 있지만, 해당 볼륨은 Docker 상에서 이미지(image)나 컨테이너(container), 네트워크(network)와 비슷한 방식으로 관리가 되는 이점이 있다. 그래서 대부분의 상황에서는 볼륨을 사용하는 것이 권장되지만 컨테이너화된 로컬 개발 환경을 구성할 때는 바인드 마운트가 더 유리할 수 있다.
로컬에서 개발을 할 때는 일반적으로 현재 작업 디렉터리에 프로젝트 저장소를 git clone 받아놓고 코드를 변경한다. 따라서 바인드 마운트를 이용해서 해당 디렉터리를 컨테이너의 특정 경로에 마운트해주면 코드를 변경할 때 마다 변경 사항을 실시간으로 컨테이너를 통해 확인할 수 있다. 반대로 컨테이너를 통해 변경된 부분도 현재 작업 디렉터리에도 반영이 되기 때문에 편리할 것이다.
```
